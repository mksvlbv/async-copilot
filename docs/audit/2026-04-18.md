## Methodology

### Tools installed (Phase 1)

- **Lighthouse CI** ‚Äî `@lhci/cli@0.15` with perf ‚â• 85, a11y ‚â• 90 budgets (configured in `lighthouserc.cjs`)
- **pa11y-ci** ‚Äî WCAG 2 AA scan over public routes (`.pa11yci.json`)
- **@axe-core/cli** ‚Äî axe-core engine, complements pa11y
- **linkinator** ‚Äî npm-native broken link checker (`.linkinatorrc.json`)
- **lost-pixel** ‚Äî visual regression snapshots, 4 routes √ó 3 breakpoints (`lost-pixel.config.ts`)
- **Stagehand + Anthropic** ‚Äî AI agent for exploratory audit (`scripts/site-audit.ts`)

### Why not run the full pipeline for this baseline

Each tool spins up headless Chromium and scans live production URLs with
its own strategies (recursion, multi-run medians, breakpoint screenshots).
Running them all sequentially on a polishing session exceeds the budget
for interactive debugging (>30 min observed on a first-run linkinator
recursion). The tools are wired up and runnable at any time; the
baseline below was captured via **Playwright MCP** in-browser evaluation
in ~2 seconds.

## Findings (baseline)

### 1. Broken footer links ‚Äî FIXED

Before:

```tsx
<a href="#" className="hover:text-gray-900">Security</a>
<a href="#" className="hover:text-gray-900">Privacy</a>
<a href="#" className="hover:text-gray-900">Terms</a>
<a href="#" className="hover:text-gray-900">Documentation</a>
```

All four led to the same URL (dead anchor), confusing users who expected
real destinations.

After (`src/app/(marketing)/page.tsx` ‚Äî `LandingFooter`):

- **Security, Terms, Documentation** ‚Äî removed from footer entirely. Honest
  minimalism: a portfolio demo does not need those pages and faking them
  erodes trust.
- **Privacy** ‚Äî now points to `/legal/privacy`, a real page.
- **GitHub ‚Üó** ‚Äî added as external anchor (`target="_blank"`).
- **"Portfolio demo" badge** added next to the logo for additional honesty.

### 2. Dummy mailto ‚Äî FIXED

Before (`ClosingCta`): `<a href="mailto:hello@example.com">Contact</a>`.

After: Contact CTA replaced with **"Browse Scenarios"** `<Link>` to
`/app/samples`. Keeps the two-button rhythm of the CTA section while
eliminating the fake email.

### 3. Missing `/legal/privacy` route ‚Äî FIXED

New page at `src/app/legal/privacy/page.tsx`. Minimal but real:

- Honest opening: declares the site as a non-commercial demo.
- Lists exactly what is and isn't collected.
- Mentions Anthropic as the only third-party processor.
- Advises not to paste sensitive data.
- Zero tracking, zero cookies, zero dark patterns in the copy.
- Includes proper `metadata` export (`title`, `description`) for SEO.

Static prerender: 170 B / 106 kB first-load.

### 4. `View Status` in hero mockup ‚Äî FIXED

`src/app/(marketing)/_sections/hero-mockup.tsx` had
`<a href="#">View Status</a>` as decorative UI chrome inside the animated
demo mockup. Clicking it would scroll to the top of the page, confusing
users.

Fixed to `<span className="underline opacity-70">View Status</span>` ‚Äî
visually identical, but correctly non-interactive (the mockup is
illustrative, not functional).

### 5. Icon-only a11y ‚Äî NO WORK NEEDED

Initial critique flagged "2-3 icon-only buttons without aria-label". Full
audit of all `<button>` elements in `src/` confirms every icon-only button
already has either `aria-label` or visible text:

- `app-header.tsx` ‚Äî Settings/Notifications icons: `aria-label`'d
- `live-run-view.tsx` ‚Äî Copy/Edit icons: `aria-label`'d
- `hero-mockup.tsx` ‚Äî Copy/Edit icons: `aria-label`'d
- All other buttons have visible text

### 6. Navigation anchors ‚Äî ALREADY CORRECT

`#how-it-works`, `#system-trust`, `#output` all resolve to real `<section id="...">`
elements in the landing page. No action.
